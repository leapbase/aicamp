Session 1: Practical Prompting Deep Dive

Part 1: Prompting Techniques Workshop (60 minutes)

Prompting is a skill, not magic. Let's break down what actually works and why.

1. Be Specific and Direct

Bad: "Write some code for me"

Good: "Write a Python function that takes a list of dictionaries and returns only those with a 'status' field equal to 'active'. Include type hints and a docstring."

Why: Vague prompts get vague results. Specificity eliminates ambiguity.

2. Provide Examples (Few-Shot Prompting)

Instead of: "Extract names from this text"

Try: "Extract person names from the text. Examples:
Input: 'Sarah called John yesterday.'
Output: ['Sarah', 'John']

Input: 'The Amazon rainforest...'
Output: []

Now extract from: [your actual text]"

Why: Examples show format and edge cases (like 'Amazon' not being a name).

3. Assign a Role (Role Prompting)

"You are an expert Python developer reviewing code for security vulnerabilities. Focus especially on SQL injection, XSS, and authentication bypasses."

Why: Roles activate relevant training data and set expectations for tone/depth.

4. Use Chain-of-Thought

"Before answering, think through the problem step by step. Show your reasoning, then give the final answer."

Why: Asking the model to "think out loud" improves reasoning, especially for complex problems.

5. Set Constraints

"Respond in exactly 3 bullet points, each under 20 words. Do not use jargon."

Why: Constraints prevent rambling and ensure output fits your use case.

Part 2: Hands-On Prompting Challenge (45 minutes)

Time to apply these techniques. You'll be given a challenging prompt scenario and iterate until you get a good result.

The Challenge:

Get the AI to write a function that solves a moderately complex problem (e.g., "Parse a CSV with error handling, validate email addresses, and return structured data"). Your first prompt will likely fail or produce incomplete code.

What You'll Learn:

  • How to diagnose why a prompt failed
  • When to add examples vs. when to be more specific
  • How to iterate on prompts (Diligence from the 4 D's!)
  • The difference between what you meant and what you said

Deliverable: Save your prompt iterations in a document. Track what you changed and why. This process is what working with AI looks like in practice.

Part 3: System Prompts — The Hidden Layer (30 minutes)

System prompts are like the "rules of engagement" for an AI conversation. They're set once at the start and influence every response.

What System Prompts Control:

  • Personality and tone: Formal vs. casual, concise vs. detailed
  • Expertise level: Expert coder vs. beginner-friendly explainer
  • Output format: Always return JSON, always include code comments, etc.
  • Safety guardrails: Don't execute code, don't make assumptions, always cite sources

Example System Prompt:

"You are a senior software engineer pair programming with a junior developer. Your responses should:
- Explain the 'why' behind decisions, not just the 'what'
- Write production-quality code with error handling
- Point out potential edge cases
- Be encouraging but honest about trade-offs"

Exercise:

Write a system prompt for your Week 1 chatbot that makes it a Python tutor for beginners. Test it with a few questions and see how it changes the responses.

Session 2: Structured Output & Tool Use

Part 1: Why Structured Output Matters (20 minutes)

So far, the AI has given you back free-form text. But what if you need reliable, parseable data?

The Problem with Free-Form Text:

  • Parsing is fragile (one extra space breaks your regex)
  • Format can vary response-to-response
  • Difficult to integrate into applications

The Solution: JSON Mode & Function Calling

JSON Mode: Forces the model to respond with valid JSON

Function Calling (Tool Use): The model can invoke functions you define, with type-safe parameters

Both OpenAI and Anthropic support these features (with slightly different syntax).

Part 2: Function Calling / Tool Use Tutorial (60 minutes)

Let's walk through how function calling works, step by step.

The Concept:

  1. You define a function schema (name, description, parameters)
  2. The model decides when to call your function
  3. The model generates a function call with arguments
  4. Your code executes the function and returns results
  5. The model uses those results to continue the conversation

Example Use Case: Weather Bot

User: "What's the weather in Seattle?"

  • Model decides it needs weather data
  • Model calls get_weather(location="Seattle")
  • Your code hits a weather API and returns JSON
  • Model says: "It's 55°F and rainy in Seattle."

Hands-On:

Implement a simple function-calling example with both OpenAI and Anthropic APIs. Use a mock function (don't worry about real APIs yet).

Key Insight

Function calling is how you give LLMs superpowers. They can search databases, call APIs, do calculations, and more—all by invoking functions you control. This is the foundation of AI agents, which you'll build later in the camp.

Part 3: Build a Real Tool — Job Posting Parser (60-90 minutes)

Now let's build something practical: a tool that extracts structured information from messy text.

The Project:

Create a CLI tool that takes a job posting (as text or a URL) and extracts:

  • Job title
  • Company name
  • Location (or "Remote")
  • Required skills (as a list)
  • Years of experience required
  • Salary range (if mentioned)

Output should be clean JSON that you could insert into a database.

Technical Requirements:

  • Use function calling or JSON mode (your choice)
  • Handle edge cases (missing fields)
  • Work with both OpenAI and Anthropic (add a --provider flag)
  • Include error handling for API failures

Stretch Goals:

  • Accept a URL and scrape the posting (use requests + BeautifulSoup)
  • Process multiple job postings in batch
  • Save results to a SQLite database or CSV file
  • Compare results between OpenAI and Anthropic—which is more accurate?

Homework

Required Tasks

  1. Finish the Building with the Claude API Course: If you haven't completed it, do so now. This course is too valuable to skip.
  2. Complete the Job Posting Parser: Make sure it works with both providers and handles edge cases.
  3. Test with Real Data: Find 5 real job postings online (from LinkedIn, Indeed, etc.) and run them through your tool. Document where it succeeds and where it fails.
  4. Write a System Prompt: Add a system prompt to your parser that makes the extraction more accurate. Experiment with different phrasings and note what works best.
  5. Push to GitHub: Update your repo with the new project. Include a README with examples and setup instructions.

Key Takeaways

  • Prompting is an iterative skill - Your first prompt rarely works perfectly; iteration is expected
  • System prompts are powerful - They let you "program" the AI's personality and behavior
  • Structured output makes AI practical - JSON mode and function calling bridge the gap between demos and production apps
  • Function calling is foundational - Almost every advanced AI application uses some form of function calling
  • Test with real data - Toy examples hide edge cases; real data reveals them
