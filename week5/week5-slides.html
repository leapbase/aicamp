Session 1: Build & Test Your Skill

1. Create the Skill File

Follow Claude Code's skill format, write clear trigger conditions, document parameters and options

2. Write the Skill Logic

Define instructions Claude should follow, specify context gathering, format output template

3. Add Examples

Include sample inputs and expected outputs, show edge cases

4. Test Thoroughly

Run on real code, try different scenarios, refine based on results

Iteration is Expected

Your first version won't be perfect. Skills often need 3-5 iterations before they're truly useful.

Session 2: Why Use Multiple Agents?

Benefits

  • Separation of concerns
  • Specialized expertise
  • Checks and balances
  • Parallel processing

When to Use

  • Complex workflows
  • Multiple perspectives needed
  • Quality requires iteration
  • Tasks can be parallelized

Key Insight

Single agents can handle simple tasks, but complex workflows benefit from specialization. Think of it like a team of experts rather than one generalist.

8 / 12

Common Orchestration Patterns

Sequential Pipeline

Agent 1 → Agent 2 → Agent 3

Each agent passes output to the next in a linear chain

Parallel + Synthesize

Multiple agents work independently, then a coordinator combines results

Iterative Refinement

Creator ⇄ Critic

Two agents iterate back and forth until quality threshold is met

Hierarchical

Manager delegates to workers and synthesizes results from the team

  • Is writing clear?
