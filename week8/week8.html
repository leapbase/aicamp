Why This Course Matters for Agents

Session 1: Agent Fundamentals & Build from Scratch

Part 1: What Is an Agent? (30 minutes)

Definition:

An agent is an AI system that can perceive its environment, make decisions, and take actions to achieve goals—autonomously and iteratively.

The Think-Act-Observe Loop:

  1. Think: Analyze the current state and decide what to do next
  2. Act: Execute an action (call an API, run a command, write a file)
  3. Observe: See the results of the action
  4. Repeat: Continue until the goal is achieved or a stopping condition is met

Agents vs. Tools:

  • Tool (e.g., Claude Code): Human-in-the-loop, step-by-step, requires approval
  • Agent: Autonomous, multi-step, minimal human intervention

Types of Agents:

  • ReAct (Reason + Act): Explains reasoning before each action
  • Plan-and-Execute: Creates full plan upfront, then executes
  • Reflexive: Reacts to environment without explicit planning
  • Hierarchical: Multiple agents with different roles

Real-World Agent Examples:

  • Code generation and debugging bots
  • Customer support automation
  • Research assistants (search + summarize + synthesize)
  • DevOps automation (monitoring + response)
  • Personal assistants (calendar + email + tasks)

Part 2: Build a Minimal Agent (90 minutes)

Let's build a simple but functional agent from scratch to understand the internals.

Project: Web Search Research Agent

An agent that answers questions by searching the web and synthesizing results.

Required Capabilities:

  • Tool 1: Web Search - Search DuckDuckGo or similar
  • Tool 2: Scrape URL - Fetch and extract text from a webpage
  • Tool 3: Answer - Return final answer to user

Implementation (in ~50 lines):

  1. Define Tools: Write functions for search, scrape, and answer
  2. Agent Loop:
    • Call LLM with available tools and conversation history
    • Parse LLM's response (which tool to use + arguments)
    • Execute the tool
    • Add result to conversation history
    • Repeat until LLM calls "answer" tool
  3. Safety Limits:
    • Max 10 iterations (prevent infinite loops)
    • Timeout after 60 seconds
    • Log all actions

System Prompt for Agent:

"You are a research agent. Your goal is to answer the user's question accurately using web search.

Available tools:

  • search(query) - Search the web, returns titles and snippets
  • scrape(url) - Fetch full text of a webpage
  • answer(text) - Return final answer to user

Process:

  1. Search for relevant information
  2. Scrape promising URLs for details
  3. Synthesize findings into a clear answer
  4. Call answer() with your response

Be thorough but efficient. Cite sources."

Test Your Agent:

Ask questions like:

  • "What are the latest features in Python 3.12?"
  • "Who won the 2025 Nobel Prize in Physics and why?"
  • "Compare the performance of React vs. Vue in 2026"

Observe the agent's process:

  • Does it search effectively?
  • Does it scrape the right URLs?
  • Are the answers accurate and well-cited?
  • How many steps does it take?

Part 3: Agent Debugging & Improvement (30 minutes)

Common Agent Failure Modes:

  • Looping: Agent repeats the same action endlessly
  • Hallucination: Agent claims to have taken actions it didn't
  • Tool Misuse: Wrong tool or malformed arguments
  • Premature Completion: Answers before gathering enough info
  • Excessive Steps: Overthinks simple questions

Debugging Techniques:

  • Log every LLM call and tool invocation
  • Add step numbers to track progress
  • Print the agent's reasoning at each step
  • Manually inspect failing cases
  • Add assertions (e.g., "scrape must receive valid URL")

Improvement Ideas:

  • Add a "reflect" step where agent critiques its own plan
  • Implement memory (save past searches to avoid redundant work)
  • Add cost tracking (tokens used)
  • Create a better stopping condition

Session 2: OpenClaw and Production Agents

Part 1: Introduction to OpenClaw (30 minutes)

What Is OpenClaw?

OpenClaw is a production-ready agent framework. It handles the infrastructure so you can focus on defining agent behavior.

Key Features:

  • Multi-Platform: Telegram, Discord, Slack, web interface
  • Skill System: Modular, reusable capabilities
  • Memory Management: Conversation history, user preferences
  • Security: Permissions, sandboxing, audit logs
  • Monitoring: Metrics, alerts, cost tracking

OpenClaw vs. Your Scratch Agent:

  • Your Agent: Educational, simple, 50 lines
  • OpenClaw: Production-grade, scalable, thousands of lines

Both use the same think-act-observe loop, but OpenClaw adds robustness, security, and integrations.

Part 2: Deploy OpenClaw (60 minutes)

Deployment Options:

  • Local: Run on your laptop (easiest for testing)
  • Free-Tier VPS: Oracle Cloud, Google Cloud free tier
  • Docker: Containerized deployment

Setup Steps:

  1. Clone and Install:
    • Clone OpenClaw repository
    • Install dependencies
    • Configure environment variables
  2. Connect to Telegram or Discord:
    • Create a bot account
    • Get API token
    • Configure OpenClaw to use it
  3. Configure Permissions:
    • What can the agent do?
    • What requires approval?
    • What's forbidden?
  4. Test Basic Functionality:
    • Send a message to your bot
    • Ask it simple questions
    • Verify it responds

Initial Testing:

Try these commands with your OpenClaw bot:

  • "What can you do?"
  • "Search for news about AI in 2026"
  • "Remind me in 30 minutes to take a break"
  • "What's the weather in San Francisco?"

Part 3: Write a Custom OpenClaw Skill (60 minutes)

Skill Ideas:

  • GitHub Integration: Create issues, check PR status
  • Note Taker: Save snippets of conversation to a file
  • Daily Briefing: Aggregate news, weather, calendar
  • Code Runner: Execute safe code snippets
  • Flashcards: Quiz user from a deck

Implementation:

  1. Create a new skill file in OpenClaw's skills directory
  2. Define the skill's capabilities (tools and resources)
  3. Write the tool handler functions
  4. Register the skill in OpenClaw's config
  5. Test thoroughly

Example: GitHub Issue Creator Skill

Lets the agent create GitHub issues on command.

  • Tool: create_github_issue(repo, title, body)
  • Implementation: Use GitHub API or PyGithub library
  • Permission: Requires user approval before creating
  • Usage: "Create an issue in my-repo for adding tests to the API"

Part 4: Agent Security Discussion (45 minutes)

Agents can take actions autonomously. This is powerful but risky. Let's think through the security implications.

Key Questions:

  • What could go wrong? What's the worst an agent could do?
  • Blast radius: If compromised, what's the damage?
  • Permissions: What should require approval vs. be automatic?
  • Prompt injection: Can a user trick the agent into unauthorized actions?
  • API keys: How do you store secrets safely?

Security Best Practices:

Critical Security Principles

  • Least Privilege: Grant minimal permissions needed
  • Human-in-the-Loop: Require approval for destructive actions
  • Audit Logging: Log every action for accountability
  • Rate Limiting: Prevent runaway loops from burning money
  • Sandboxing: Isolate agent execution environment
  • Input Validation: Sanitize all user input

Permission Levels:

  • Auto-approve: Safe, read-only operations (search, read files)
  • Require approval: Write operations (create files, send messages)
  • Forbidden: Destructive operations (delete, deploy to production)

Threat Modeling Exercise:

For your OpenClaw deployment:

  1. List all capabilities your agent has
  2. For each, describe a misuse scenario
  3. Propose a mitigation (permission, approval, logging)
  4. Determine appropriate permission level

Thinking About Blast Radius

Blast radius is the maximum damage an agent could cause. Always ask: "If this agent were fully compromised, what's the worst that could happen?" Design your permissions accordingly.

Key Takeaways

  • Agents are autonomous AI systems - They think, act, and observe in loops
  • The think-act-observe loop is fundamental - Applies to all agent architectures
  • Agents are powerful but risky - Security and permissions are critical
  • Building from scratch teaches internals - But production frameworks like OpenClaw are better for real use
  • Blast radius thinking is essential - Always consider worst-case scenarios
  • Human-in-the-loop is often wise - Especially for destructive actions
  • Agent design is about trade-offs - Autonomy vs. safety, speed vs. accuracy, cost vs. thoroughness
