5 / 12

Quality Over Speed

A Polished 80% Beats a Sloppy 100%

It's tempting to rush. Don't. A well-built, partially complete project is better than a buggy, complete one.

Code Quality Checklist

  • Handles errors gracefully?
  • Clear variable/function names?
  • Complex logic commented?
  • Another developer could understand?
  • Tested?

When to Refactor

  • Copy-pasting code? Extract function
  • Function > 50 lines? Split it
  • Confused by own code? Refactor now
  • Tests hard to write? Code too complex

Managing Technical Debt

Technical Debt Is Real

Every shortcut you take now, you'll pay back with interest later. For a 3-week project, some debt is okay - but track it.

Create GitHub Issues

Document "known issues" and "future improvements" so interviewers see you understand trade-offs

Be Honest in Documentation

A "Known Limitations" section shows maturity and self-awareness

Open Source Contribution

Why Contribute?

  • Give back to tools you use
  • Great for your resume
  • Learn from code reviews
  • Build your network

Target Repos

  • AI tooling you've used
  • Claude Code, MCP servers
  • RAG frameworks, agent libraries
  • Look for "good first issue"

Types of Contributions

Bug fixes (high impact, often easier), documentation improvements, example code or tutorials, new features (if clearly needed)

8 / 12

Making Your Contribution

1. Find and Claim Issue

Comment on issue with your approach. Wait for maintainer response.

2. Fork and Branch

Fork the repo, create a branch with clear name

3. Implement Fix/Feature

Understand codebase, match style, test thoroughly

4. Create PR with Clear Description

Explain problem, solution, testing. Reference issue number.

5. Respond to Feedback

Be prompt, gracious, and willing to make changes

Productivity Tips

Environment

  • Remove distractions
  • Dedicated workspace
  • Good music or silence
  • Stay hydrated, eat well

Time Management

  • Time-box features
  • Track your time
  • Schedule breaks
  • End-of-day review

When Stuck

  • First 10 min: Read errors, Google
  • Next 20 min: Try different approaches
  • After 30 min: Ask Claude Code
  • After 60 min: Ask a human

Celebrate Progress

  • Close issue? Take 5-min break
  • Finish feature? Share screenshot
  • Hit milestone? Post on social
  • Momentum builds on visibility
