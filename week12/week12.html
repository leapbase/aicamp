Session 1: Resume Review & LinkedIn Optimization

Part 1: Resume Structure for AI Roles (45 minutes)

Resume Sections (in order):

1. Header:

  • Name (large, prominent)
  • Email, Phone, LinkedIn, GitHub
  • Location (city, state) - optional
  • Personal website or portfolio (if you have one)

2. Education:

  • University, degree, expected graduation
  • GPA (if 3.5+)
  • Relevant coursework (CS, math, AI courses)

3. Skills:

  • Languages: Python, JavaScript, SQL, etc.
  • AI/ML: Claude API, OpenAI API, RAG, embeddings, vector databases, prompt engineering
  • Tools & Frameworks: Claude Code, Git, Docker, Streamlit, FastAPI, React, etc.
  • Concepts: Agent orchestration, MCP, function calling, system design

4. Projects (Most Important Section):

For each project (list 3-4 best):

  • Project Name | Tech Stack | [GitHub] [Live Demo]
  • 1-2 sentence description focusing on impact/purpose
  • 2-3 bullets with specific accomplishments:
    • Start with action verbs (Built, Implemented, Designed, Deployed)
    • Include metrics where possible (reduced latency by 200ms, processed 10k documents)
    • Highlight technical decisions and challenges solved

5. Certifications:

  • List all 9-10 Anthropic course certificates
  • Include completion dates
  • Link to verification if possible

6. Experience (if applicable):

  • Previous internships, research, relevant jobs
  • Same bullet format as projects

Example Project Bullet:

ArXiv Research Assistant MCP Server | Python, MCP SDK, Claude API, PyPDF2 | [GitHub] [Demo]

Custom Model Context Protocol server enabling Claude to search, download, and analyze academic papers from arXiv

  • Implemented PDF parsing and summarization pipeline processing 100+ papers with 95% accuracy
  • Designed MCP resource and tool handlers with rate limiting and caching, reducing API calls by 60%
  • Integrated with Claude Code allowing natural language research queries (e.g., "compare transformer architectures")

Resume Dos:

  • Keep it to 1 page (as a student)
  • Use consistent formatting
  • Quantify achievements wherever possible
  • Tailor for each application (emphasize relevant skills)
  • Use keywords from job descriptions
  • Proofread obsessively (zero typos)

Resume Don'ts:

  • Generic objective statements
  • Listing courses without context
  • Vague bullets ("worked on AI project")
  • Including high school activities (unless truly exceptional)
  • Lying or exaggerating

Session 2: Mock Interviews & Demo Day

Part 1: Technical Interview Prep (60 minutes)

Common Interview Question Categories:

1. Project Deep-Dive:

"Tell me about your capstone project."

  • Describe the problem and why it matters
  • Explain your architecture and tech choices
  • Discuss challenges and how you solved them
  • Share metrics or results
  • Mention what you'd improve

2. Technical Knowledge:

  • "What's the difference between embeddings and fine-tuning?"
  • "Explain how RAG works."
  • "What is prompt engineering? Give me an example."
  • "How do LLM APIs work? What's a token?"
  • "What's MCP and why does it matter?"

3. Tools & Workflow:

  • "What AI tools do you use and when?"
  • "How has Claude Code changed your development workflow?"
  • "Describe your process for debugging AI-generated code."

4. Design Questions:

  • "Design a chatbot for customer support."
  • "How would you build a code search tool for a large codebase?"
  • "Design a system that monitors and alerts on AI agent behavior."

5. Behavioral:

  • "Tell me about a time you got stuck on a hard problem."
  • "Describe a technical decision you made and why."
  • "What's a project you're proud of and why?"

Interview Answering Framework (STAR):

  • Situation: Set the context
  • Task: What was the goal or challenge?
  • Action: What did you do specifically?
  • Result: What was the outcome? Metrics if possible.

Practice Questions for Mock Interviews:

  1. "Walk me through your capstone project from start to finish."
  2. "What's the most interesting technical problem you solved this summer?"
  3. "How would you explain RAG to a non-technical person?"
  4. "What are the limitations of current LLMs?"
  5. "Design an AI-powered code review tool."
  6. "Why are you interested in working with our company?"

Pair up and practice. Give each other feedback.

Part 2: Capstone Demo Presentations (2-3 hours)

Each student presents their capstone to the group.

Presentation Format (5 minutes each):

  1. Introduction (30 sec):
    • Name and project name
    • One-sentence description
  2. Problem & Motivation (45 sec):
    • What problem does this solve?
    • Why does it matter?
  3. Demo (2-3 min):
    • Show the application working
    • Highlight 3-5 key features
    • Use interesting, real examples
  4. Technical Highlights (1 min):
    • Architecture overview
    • Key technologies
    • Interesting technical decisions
  5. Lessons & Future (30 sec):
    • What you learned
    • What's next for the project
  6. Q&A (2-3 min):
    • Audience asks questions
    • Answer clearly and honestly

Presentation Tips:

  • Practice your timing (don't go over 5 minutes for the demo)
  • Speak clearly and at a moderate pace
  • Make eye contact (don't just stare at screen)
  • Show enthusiasm for your project
  • Be prepared for "What would you do differently?" questions
  • Have your demo ready to go (don't troubleshoot live)

Peer Feedback:

After each presentation, the group provides:

  • One thing that was impressive
  • One question or curiosity
  • One suggestion for improvement

This is great practice for real demos and interviews.
