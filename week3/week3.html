Session 1: Claude Code Setup & Contributing to Open Source

Part 1: Claude Code Environment Setup (45 minutes)

Installation Process:

  1. Install Claude Code CLI:
    • Download from Anthropic's website
    • Configure with your API key
    • Test basic commands
  2. VS Code Integration:
    • Install the Claude Code extension
    • Learn keyboard shortcuts
    • Configure settings for your workflow
  3. Git Integration:
    • Understand how Claude Code uses Git
    • Best practices for commits with AI assistance
    • How to review AI-generated diffs

Create Your First CLAUDE.md:

CLAUDE.md is a project documentation file that tells Claude Code about your project. Think of it as a system prompt for your codebase.

What to include in CLAUDE.md:

  • Project overview and purpose
  • Architecture and key components
  • Coding conventions and style guide
  • Common commands and setup instructions
  • Areas where Claude should be cautious
  • Testing requirements

Create a CLAUDE.md for your Week 1 chatbot project as practice.

Part 2: Essential Slash Commands & Plan Mode (30 minutes)

Core Slash Commands:

  • /edit - Make targeted changes to files
  • /add - Create new files or add features
  • /ask - Ask questions about the codebase
  • /debug - Help diagnose and fix issues
  • /test - Generate or run tests
  • /plan - Create a multi-step plan before executing

Understanding Plan Mode:

Plan mode is one of Claude Code's most powerful features. Instead of diving straight into changes, Claude:

  1. Analyzes the request and codebase
  2. Proposes a multi-step plan
  3. Waits for your approval
  4. Executes the plan step by step
  5. Checks with you before major changes

When to use plan mode: Complex features, refactoring, anything that touches multiple files

When to skip it: Small bug fixes, single-file changes, simple additions

Part 3: Contributing to Open Source (60-75 minutes)

Now let's use Claude Code on a real open-source project. This is a milestone—you'll make a genuine contribution to a real codebase.

Step 1: Find a Good First Issue

Look for repositories with "good first issue" or "beginner-friendly" labels. Ideal projects:

  • Active maintenance (recent commits)
  • Clear contribution guidelines
  • Small, focused tools or libraries
  • Written in Python or JavaScript

Suggested repos to explore:

  • VS Code extensions with small feature requests
  • CLI tools that need documentation or bug fixes
  • Python libraries with feature requests tagged "help wanted"

Step 2: Set Up the Project

  1. Fork the repository
  2. Clone your fork locally
  3. Read CONTRIBUTING.md (if it exists)
  4. Create a CLAUDE.md based on the project README
  5. Run tests to ensure everything works

Step 3: Use Claude Code to Add a Feature

Prompt Claude Code to implement the feature or fix the bug. Example workflow:

Your prompt: "I want to add a feature that [describe feature]. This addresses issue #123. Please create a plan first."

Claude creates plan: Review it carefully. Does it make sense? Are there edge cases?

Approve and execute: Let Claude implement the changes

Test manually: Run the code yourself, try edge cases

Run automated tests: Make sure nothing broke

Review the diff: Read every line Claude changed

Step 4: Submit a Pull Request

  • Commit with a clear message
  • Push to your fork
  • Open a PR against the upstream repo
  • Write a description explaining what you changed and why
  • Link to the original issue

Important: You're Still Responsible

Claude Code wrote the code, but you're the one submitting the PR. You must understand every change and be ready to defend it in code review. This is Discernment from the 4 D's in action.

Session 2: Build Something from Scratch

Part 1: Project Selection & Planning (20 minutes)

You'll build a complete small application from scratch using Claude Code. Choose one:

Option 1: Personal Dashboard CLI

A command-line tool that shows you daily information:

  • Weather for your location
  • Top Hacker News stories
  • GitHub activity from repos you follow
  • Calendar events (from a CSV or JSON file)

Option 2: Markdown Blog Generator

A static site generator that turns Markdown files into a blog:

  • Parses Markdown files with frontmatter (title, date, tags)
  • Generates HTML pages with a template
  • Creates an index page and tag pages
  • Includes basic CSS styling

Option 3: Code Snippet Manager

A CLI tool for saving and retrieving code snippets:

  • Save snippets with tags and descriptions
  • Search by tag or keyword
  • Copy snippets to clipboard
  • Store in SQLite database

Or propose your own idea of similar scope—something you'd actually use.

Part 2: Initial Setup with Claude Code (30 minutes)

Start with a Plan:

Use plan mode to scope the project. Prompt Claude Code:

"I want to build a [project description]. It should have these features: [list features]. Create a detailed plan including file structure, key functions, and implementation steps."

Review and Refine the Plan:

  • Is the scope reasonable? (Can you finish in one week?)
  • Are there missing edge cases?
  • Does the file structure make sense?
  • Are dependencies reasonable?

Create the Project Structure:

Let Claude Code set up:

  • Directory structure
  • README.md with project description
  • requirements.txt or package.json
  • CLAUDE.md with project context
  • .gitignore
  • Basic entry point file

Part 3: Iterative Development (90 minutes)

Now build the application feature by feature. This is where you learn how to work effectively with an AI coding partner.

Feature-by-Feature Approach:

  1. Start with Core Functionality: Get the minimal viable version working first
  2. Test Immediately: After each feature, run it manually. Does it work?
  3. Iterate on Prompts: If Claude's first attempt fails, don't start over—refine your prompt
  4. Add Error Handling: Once features work, make them robust
  5. Improve UX: Add helpful messages, colors, formatting

Prompting Techniques for This Phase:

Good prompts:

  • "Implement the weather fetching feature. Use the OpenWeather API. Handle network errors gracefully."
  • "The current output is messy. Add color-coded formatting using the rich library."
  • "Add command-line arguments using argparse. Support --help, --verbose, and --config flags."

Less effective prompts:

  • "Make it better" (too vague)
  • "Fix everything" (no direction)
  • "Add all the features at once" (too much at once)

When to Intervene:

Know when to step in manually:

  • Debugging: If Claude keeps making the same mistake, fix it yourself
  • Critical Logic: Security-sensitive code deserves manual review
  • Performance: If code works but is slow, profile it yourself
  • Dependencies: Verify that suggested libraries are maintained and trustworthy

Homework

Required Tasks

  1. Complete Your Application: Finish the project you started in Session 2. It should be fully functional with error handling.
  2. Write Comprehensive Documentation:
    • README with installation and usage instructions
    • Screenshots or demo GIF showing it in action
    • CLAUDE.md explaining the architecture
    • Code comments for complex functions
  3. Follow Up on Your Open Source PR: Respond to any code review feedback. If maintainers suggest changes, use Claude Code to implement them.
  4. Continue Claude Code in Action: Watch at least 2 more chapters of the course.
  5. Push Everything to GitHub: Both your new app and your open-source contribution fork.

Key Takeaways

  • Claude Code is a collaborator, not a magic button - You still need to guide it, review its work, and intervene when needed
  • CLAUDE.md is your project's system prompt - Good context leads to better suggestions
  • Plan mode prevents chaos - For complex tasks, always review the plan before execution
  • Open-source contribution is valuable - Even small PRs show you can work with real codebases
  • Testing is still your job - AI can write tests, but you must verify the code actually works
  • Iteration is normal - Your first prompt won't be perfect; refining is part of the process
