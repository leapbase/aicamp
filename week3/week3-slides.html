From APIs to Agents

A Major Paradigm Shift

Weeks 1-2 taught you to call AI APIs directly. Now you're moving to an agentic tool—Claude Code can read files, edit code, run commands, and iterate on its own work. This is your first exposure to AI as a collaborator, not just a function you call.

API Approach

  • You control every step
  • Single request/response
  • Manual iteration
  • Limited context

Agent Approach

  • AI takes initiative
  • Multi-step workflows
  • Self-correction
  • Deep codebase understanding

Your First CLAUDE.md

Project Documentation as System Prompt

CLAUDE.md tells Claude Code about your project. Think of it as a system prompt for your codebase.

What to Include

  • Project overview and purpose
  • Architecture and key components
  • Coding conventions and style
  • Common commands and setup

Best Practices

  • Areas to be cautious
  • Testing requirements
  • Known issues or limitations
  • Deployment instructions

Essential Slash Commands

Core Commands

  • /edit - Make targeted changes
  • /add - Create new files or features
  • /ask - Ask about the codebase
  • /debug - Diagnose and fix issues

Advanced Commands

  • /test - Generate or run tests
  • /plan - Create multi-step plans
  • /review - Code review
  • /docs - Generate documentation

Pro Tip: Use Plan Mode

For complex features, use /plan to review Claude's approach before execution. This prevents surprises and ensures you're aligned.

7 / 12

Understanding Plan Mode

1. Analyze

Claude examines your request and the codebase context

2. Propose

Creates a detailed multi-step plan with rationale

3. Approve

You review the plan and can request changes before execution

4. Execute

Claude implements the plan step by step, checking in before major changes

When to Use

  • Complex features
  • Major refactoring
  • Multi-file changes

When to Skip

  • Simple bug fixes
  • Single-file changes
  • Quick additions

Contributing to Open Source

Find a Good First Issue

Look for beginner-friendly labels, active maintenance, clear contribution guidelines, Python or JavaScript projects

Set Up the Project

Fork, clone, read CONTRIBUTING.md, create CLAUDE.md, run tests

Use Claude Code

Create a plan first, implement changes, test manually and automated, review every line

Submit a Pull Request

Clear commit message, push to fork, open PR with description, link to original issue
