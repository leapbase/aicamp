What Is Model Context Protocol?

MCP is USB for AI

A standardized protocol for connecting AI models to external data sources and tools. It's a universal interface that lets Claude connect to anything.

Why MCP Exists

  • Solves integration problem at scale
  • Standardized across applications
  • Reusable and composable
  • Discoverable capabilities

Key Components

  • MCP Servers (expose data/tools)
  • MCP Clients (connect to servers)
  • Resources (read-only data)
  • Tools (actions to perform)

MCP Architecture & How It Works

The Model

  • Server exposes resources and tools
  • Client connects to servers
  • Resources: read-only data
  • Tools: actions AI can perform
  • Prompts: pre-defined templates

The Flow

  • Configure Claude Code to connect
  • Claude discovers capabilities
  • When relevant, reads resources
  • When needed, invokes tools
  • Results flow back to Claude

MCP vs Function Calling

Function calling is one-off and custom. MCP is standardized, reusable, discoverable, and composable. MCP is function calling + a universal protocol.

Built-In MCP Servers

Filesystem MCP

Gives Claude access to your filesystem with permissions.

  • Read files and directories
  • Search by name or content
  • Watch for changes
  • Create, edit, delete (with approval)

GitHub MCP

Connects Claude to GitHub repositories.

  • List repos, issues, pull requests
  • Read files from any branch
  • Create issues and PRs
  • Comment on PRs
  • Search code across repos

Database MCP

Connects to PostgreSQL, MySQL, or SQLite.

  • Run SELECT queries
  • Describe schema
  • Execute writes (with permission)

The Power

No integration code needed - just connect servers. MCP handles the plumbing.

Building Your MCP Server

1. Define the Manifest

Decide what resources and tools your server will expose

2. Implement Resource Handlers

Write functions that fetch and format data - handle errors gracefully

3. Implement Tool Handlers

Write functions that perform actions - validate parameters and return status

4. Add Configuration

API keys from environment variables, rate limiting, logging

5. Test Independently

Use MCP inspector tool before connecting to Claude Code

6. Connect to Claude Code

Configure and verify Claude can discover and use your server
