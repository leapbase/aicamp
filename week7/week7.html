Pre-Work (Before Session 1)

Course: Introduction to Model Context Protocol

MCP is one of the most important technologies you'll learn at camp. It's a major 2026 focus area for Anthropic and a differentiator on your resume.

What MCP Is:

A standardized protocol for connecting AI models to external data sources and tools. Think of it as USB for AI—a universal interface that lets Claude connect to anything.

What You'll Learn:

  • MCP Architecture: Servers, clients, resources, tools
  • Why MCP Exists: Solving the integration problem at scale
  • Server Capabilities: What MCP servers can expose
  • Security Model: Permissions, sandboxing, trust
  • Real-World Applications: CRM integration, database access, API connections

Time commitment: 90-120 minutes

Why MCP Is Resume Gold

MCP is new (2025) and strategically important to Anthropic. Most developers don't know it yet, which makes you an early adopter. In interviews, being able to say "I built custom MCP servers" signals that you're ahead of the curve and understand modern AI architecture.

Session 1: Using Existing MCP Servers

Part 1: MCP Fundamentals (30 minutes)

The MCP Model:

  • MCP Server: Exposes resources (data) and tools (actions)
  • MCP Client: Connects to servers (e.g., Claude Code)
  • Resources: Read-only data (files, database records, API responses)
  • Tools: Actions the AI can perform (create file, send email, run query)
  • Prompts: Pre-defined interaction templates

How It Works:

  1. You configure Claude Code to connect to MCP servers
  2. Claude discovers available resources and tools
  3. When relevant to a task, Claude can:
    • Read resources (e.g., fetch a file)
    • Invoke tools (e.g., commit to git)
  4. Results flow back to Claude, which continues the task

Comparison to Function Calling:

Function calling (Week 2) and MCP serve similar purposes, but:

  • Function Calling: One-off, custom implementations per app
  • MCP: Standardized, reusable, discoverable, composable

MCP is function calling + a universal protocol.

Part 2: Connect to Built-In MCP Servers (60 minutes)

Anthropic provides several official MCP servers. Let's connect Claude Code to them.

Filesystem MCP Server:

Gives Claude access to your filesystem with permissions.

Capabilities:

  • Read files and directories
  • Search for files by name or content
  • Watch for file changes
  • Create, edit, delete files (with permission)

Hands-On Exercise:

  1. Configure the filesystem MCP server in Claude Code
  2. Set permissions (read-only for now)
  3. Ask Claude: "Find all Python files in this project that import 'requests'"
  4. Ask Claude: "What's the total line count of my codebase?"

GitHub MCP Server:

Connects Claude to GitHub repositories.

Capabilities:

  • List repositories, issues, pull requests
  • Read file contents from any branch
  • Create issues and PRs
  • Comment on PRs
  • Search code across repos

Hands-On Exercise:

  1. Connect the GitHub MCP server with a personal access token
  2. Ask Claude: "List all open issues in my AI Camp repos"
  3. Ask Claude: "Create a GitHub issue for adding tests to my Week 6 chatbot"
  4. Ask Claude: "Show me the README from my most recent project"

Database MCP Server (Optional):

Connects to PostgreSQL, MySQL, or SQLite.

Capabilities:

  • Run SELECT queries
  • Describe schema
  • Execute CREATE, UPDATE, DELETE (with permission)

Hands-On Exercise:

  • Connect to a SQLite database from one of your projects
  • Ask Claude: "What tables exist in this database?"
  • Ask Claude: "Show me the 10 most recent records"

The Power of MCP

Notice how you didn't write integration code—you just connected servers. MCP handles the plumbing, letting you focus on what you want Claude to do, not how to wire everything together.

Session 2: Build Your Own MCP Server

Part 1: MCP Server Architecture (30 minutes)

Server Components:

  • Server Manifest: Declares capabilities (resources, tools, prompts)
  • Resource Handlers: Functions that return data
  • Tool Handlers: Functions that perform actions
  • Initialization: Setup, authentication, configuration

MCP Server Lifecycle:

  1. Client (Claude Code) discovers server
  2. Server sends manifest (list of capabilities)
  3. Client invokes resources/tools as needed
  4. Server executes and returns results
  5. Client presents results to Claude

Development Environment:

Install the MCP Python SDK:

  • pip install mcp
  • Review the SDK documentation
  • Examine example servers from Anthropic

Part 3: Implement Your MCP Server (90-120 minutes)

Step 1: Define the Manifest

What will your server expose?

Example: Weather MCP Server

Resources:

  • weather/current/{location} - Current weather
  • weather/forecast/{location} - 5-day forecast

Tools:

  • get_weather - Fetch weather for any location
  • compare_weather - Compare weather between cities

Step 2: Implement Resource Handlers

Write functions that return data:

  • Fetch from APIs or databases
  • Format as JSON or structured text
  • Handle errors gracefully
  • Cache when appropriate

Step 3: Implement Tool Handlers

Write functions that perform actions:

  • Validate parameters
  • Execute the action
  • Return success/failure status
  • Log operations

Step 4: Add Configuration

  • API keys (from environment variables)
  • Rate limiting settings
  • Default parameters
  • Logging configuration

Step 5: Test Independently

Before connecting to Claude Code:

  • Use the MCP inspector tool to test your server
  • Verify all resources return valid data
  • Test all tools with various inputs
  • Handle edge cases and errors

Step 6: Connect to Claude Code

  • Configure Claude Code to use your local MCP server
  • Verify Claude can discover your resources and tools
  • Give Claude a task that requires your server
  • Debug any issues

Development Tips

  • Start with one resource and one tool—expand later
  • Use print statements or logging to debug
  • Test each component independently before integration
  • Read the MCP SDK docs carefully—they're well-written

Key Takeaways

  • MCP is a universal integration layer - Standardized, reusable, composable
  • Servers expose resources and tools - Data (read) and actions (write)
  • MCP eliminates integration boilerplate - Focus on what, not how
  • Security is built-in - Permissions, sandboxing, user control
  • Building MCP servers is high-leverage - One server = reusable across many contexts
  • This knowledge is rare and valuable - Most developers haven't heard of MCP yet
  • MCP is the future of AI integration - Understanding it puts you ahead of the curve
