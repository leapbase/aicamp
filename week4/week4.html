Session 1: OpenAI Codex CLI & Comparison Matrix

Part 1: OpenAI Codex CLI Setup & Exploration (60 minutes)

Installation & Configuration:

  1. Install the Codex CLI:
    • Follow OpenAI's installation instructions
    • Configure with your OpenAI API key
    • Test basic functionality
  2. Learn Core Commands:
    • Code generation and editing
    • Project context management
    • File navigation and search
  3. Try Common Workflows:
    • Adding a feature to existing code
    • Debugging an error
    • Refactoring for readability
    • Writing tests

Hands-On Exercise:

Use Codex CLI to accomplish the same tasks you did with Claude Code in Week 3:

  • Add a new feature to your Week 3 project
  • Fix a bug (introduce one if needed)
  • Refactor a function for better clarity

Take notes on differences you notice:

  • How do the UX and workflows differ?
  • Which tool gives better suggestions?
  • Which is faster or more efficient?
  • How do they handle context and memory?

Part 2: Build a Comparison Matrix (60 minutes)

Now that you've used multiple AI coding tools, create a structured comparison. This is valuable both for your learning and for future reference.

Tools to Compare:

  • Claude Code
  • OpenAI Codex CLI
  • GitHub Copilot (if you have access)
  • Raw API calls (OpenAI & Anthropic)

Dimensions to Evaluate:

Dimension What to Evaluate
Setup Complexity How easy is it to get started?
Code Quality How good are the suggestions?
Context Awareness How well does it understand your project?
Speed Latency of responses
Cost API pricing or subscription fees
Iteration How easy is it to refine and correct?
Best Use Cases When would you choose this tool?

Deliverable:

Create a Markdown document or spreadsheet with your findings. Include:

  • Ratings (1-5 stars) for each dimension
  • Specific examples supporting your ratings
  • Recommendations for when to use each tool
  • Personal preferences and why

Why This Matters

In interviews, you'll be asked "What AI tools do you use?" A nuanced answer—comparing tools with specific examples—shows depth. Generic answers like "I use ChatGPT" don't differentiate you. This comparison matrix is your cheat sheet for thoughtful responses.

Session 2: Codex Cloud & Long-Running Delegation

Part 1: OpenAI Codex Cloud (45 minutes)

The cloud-based coding assistant can handle longer, more complex tasks than CLI tools. Let's explore its capabilities.

Key Differences from CLI:

  • Longer Context Windows: Can handle larger codebases
  • Persistent Sessions: Maintains state across interactions
  • Async Execution: Can work on tasks while you do other things
  • Collaboration Features: Share sessions with team members

Setting Up a Cloud Session:

  1. Create a new project in the web interface
  2. Upload or connect your repository
  3. Configure project context and guidelines
  4. Test with a simple task

Use Cases for Cloud vs. CLI:

Use CLI when: Quick edits, local development, tight feedback loop needed

Use Cloud when: Large refactors, research tasks, complex features spanning multiple files

Part 2: Delegate a Complex Task (90 minutes)

This is your first experience with true task delegation. Give the AI a substantial task and let it work relatively independently.

Task Options (Choose One):

Option 1: Full-Stack Web App

  • A simple todo app with frontend and backend
  • React or Vue frontend
  • Flask or Express backend
  • SQLite or MongoDB database
  • Basic authentication

Option 2: Data Analysis Pipeline

  • Fetch data from a public API (e.g., weather, stocks, sports)
  • Clean and transform the data
  • Generate visualizations with matplotlib or plotly
  • Create a daily report as HTML or PDF
  • Schedule with cron or similar

Option 3: CLI Tool with TUI

  • A terminal user interface (TUI) application
  • File manager, system monitor, or dev tool
  • Use libraries like rich, textual, or blessed
  • Keyboard shortcuts and navigation
  • Configuration file support

Delegation Strategy:

  1. Write a Detailed Spec: Describe the project in detail (2-3 paragraphs)
    • Features and requirements
    • Tech stack preferences
    • Edge cases to handle
    • Testing requirements
  2. Set Checkpoints: "After completing the basic structure, show me the code for review before proceeding."
  3. Let It Work: Give the AI time to generate code. This might take 30-60 minutes of AI work.
  4. Review & Iterate: Test the result. What works? What doesn't? Provide feedback and iterate.

Learning Objective: Effective Delegation

This exercise teaches the Delegation dimension of the 4 D's. You're learning:

  • What level of detail to provide in specs
  • When to step in vs. let the AI continue
  • How to evaluate whether delegation succeeded
  • How to recover when the AI goes off track

Part 3: Discussion — When to Use Each Tool (30 minutes)

Based on your experience this week, discuss decision criteria for tool selection.

Discussion Framework:

For Quick Tasks (< 30 min):

  • What's the fastest tool to get a result?
  • Does quality matter, or is speed paramount?

For Learning:

  • Which tool helps you understand code better?
  • Which explains its reasoning most clearly?

For Production Code:

  • Which produces the most maintainable code?
  • Which is best for security-critical code?

For Collaboration:

  • Which tools work best in a team setting?
  • How do you share AI-generated code with teammates?

Homework

Project 1 Due: Polished CLI Tool or Web App

This is your first major deliverable. Choose either your Week 3 application or your Week 4 delegation project, and polish it to portfolio quality.

Requirements:

  1. Functionality:
    • All features work without errors
    • Edge cases are handled gracefully
    • User experience is smooth
  2. Code Quality:
    • Clean, readable code
    • Consistent style and formatting
    • Comments for complex logic
    • No obvious bugs or security issues
  3. Documentation:
    • Comprehensive README with screenshots/demo
    • Installation and setup instructions
    • Usage examples
    • Architecture overview
    • CLAUDE.md or equivalent
  4. Testing:
    • Manual testing completed
    • Automated tests (at least basic ones)
    • Test results documented
  5. Reflection:
    • Add a "Development Process" section to README
    • Describe which AI tools you used and how
    • What worked well? What was challenging?
    • What would you do differently next time?

Additional Tasks:

  • Finalize Your Comparison Matrix: Clean up and format your AI tools comparison
  • Prepare a 5-Minute Presentation: Demo your project and explain your design decisions
  • Update Your GitHub: Ensure your profile showcases your best work

Key Takeaways

  • No single tool is best for everything - Each AI coding tool has strengths and ideal use cases
  • Delegation requires clear specs - The better your initial description, the better the AI's output
  • Long-running tasks need checkpoints - Don't let AI work for hours without review
  • Comparison builds expertise - Using multiple tools makes you fluent in AI-assisted development
  • Polish matters for portfolios - A working prototype isn't enough; make it presentable
  • Documentation is part of the project - Good docs make your work accessible and impressive
