Pre-Work (Before Session 1)

Complete these two foundational courses before your first session. Both are short and specifically designed for students at your level.

Course 1: Claude 101

A quick introduction to Claude and its capabilities. This course covers:

  • What Claude is and how it differs from other AI models
  • Basic interaction patterns and best practices
  • Understanding Claude's strengths and limitations
  • Safety and responsible AI use

Time commitment: 30-45 minutes

Course 2: AI Fluency for Students

This course introduces the 4 D's framework that will guide your work all summer:

  • Delegation: Knowing what tasks to give to AI and what to do yourself
  • Description: Writing clear prompts and instructions
  • Discernment: Evaluating AI output critically
  • Diligence: Iterating and refining your approach

This framework becomes a shared vocabulary you'll use throughout the camp to discuss your AI interactions.

Time commitment: 45-60 minutes

Why These Courses Matter

These aren't just checkboxes—they establish a mental model for how to work with AI effectively. The 4 D's framework from AI Fluency for Students will come up repeatedly as you learn to delegate coding tasks, describe features clearly, discern good code from bad, and diligently iterate on your projects.

Session 1: Environment Setup & AI Fluency Discussion

Part 1: Development Environment Setup (60-90 minutes)

We'll build a complete AI development environment from scratch. By the end, you'll have everything needed to call APIs and build AI applications.

Tools to Install:

  1. Python 3.10+ - The primary language for AI development
    • Install via python.org or Homebrew (Mac) / Chocolatey (Windows)
    • Verify: python --version
  2. VS Code - Your code editor
    • Install from code.visualstudio.com
    • Key extensions: Python, Pylance, Jupyter
  3. Git - Version control (essential for collaboration)
    • Install from git-scm.com
    • Configure: git config --global user.name "Your Name"
    • Create a GitHub account if you don't have one
  4. pip & virtual environments
    • Learn to create isolated Python environments
    • Practice: python -m venv myenv

API Setup:

  1. OpenAI API Access
    • Create account at platform.openai.com
    • Add payment method (set billing cap at $10 to start)
    • Generate API key
    • Store securely using environment variables
  2. Anthropic API Access
    • Create account at console.anthropic.com
    • Add payment method (set billing cap)
    • Generate API key
    • Store in environment variables

Security Best Practice: Environment Variables

Never hardcode API keys in your code! Learn to use .env files and the python-dotenv package. Add .env to your .gitignore immediately—this prevents accidentally committing secrets to GitHub.

Part 2: AI Fluency Framework Discussion (30-45 minutes)

Now that you've completed the AI Fluency for Students course, let's apply the 4 D's to your camp work:

Discussion Questions:

  • Delegation: What programming tasks do you think AI can handle? Which ones should you always do yourself?
  • Description: How is writing a prompt similar to writing a function docstring? What makes a good description?
  • Discernment: How will you verify that AI-generated code actually works and is secure?
  • Diligence: When AI gives you wrong code, what's your debugging process?

These questions have no single right answer—they're frameworks for thinking about your work. You'll develop your own answers through practice.

Session 2: First API Calls & Build a Chatbot

Part 1: Making Your First API Calls (45-60 minutes)

Let's make our first calls to both OpenAI and Anthropic APIs. You'll see the similarities and differences.

OpenAI SDK Example:

Install the SDK: pip install openai

Simple chat completion:

  • Import the library and set up authentication
  • Create a chat completion with GPT-4
  • Parse and display the response
  • Understand tokens and pricing

Anthropic SDK Example:

Install the SDK: pip install anthropic

Equivalent call with Claude:

  • Compare API structure to OpenAI
  • Use Claude Sonnet or Opus
  • Note differences in parameters (system prompts, max_tokens)
  • Compare response formats

Hands-On Exercise:

Write a script that:

  1. Prompts the user for input
  2. Sends it to both OpenAI and Anthropic
  3. Displays both responses side-by-side
  4. Counts tokens used by each

Part 2: Build a CLI Chatbot (60-90 minutes)

Now let's build something more substantial: a command-line chatbot with conversation history.

Required Features:

  • Conversation Loop: Accept user input continuously until they type "exit"
  • Message History: Maintain context across multiple turns
  • Model Selection: Let user choose OpenAI or Anthropic at startup
  • Basic Error Handling: Handle API errors gracefully
  • Token Counter: Display tokens used per conversation

Stretch Goals:

  • Add system prompt customization
  • Implement conversation saving/loading
  • Add color-coded output (user vs. AI)
  • Show streaming responses character-by-character

Build this together in session, then customize it for homework.

Homework

Required Tasks

  1. Customize Your Chatbot: Add at least two features from the stretch goals list, or invent your own improvements.
  2. Test Both APIs: Have at least 3 conversations with your chatbot using each API. Note which responses you prefer and why.
  3. Create a GitHub Repo: Initialize a Git repository for your chatbot, add a README with setup instructions, and push to GitHub.
  4. Write a Reflection: In your README, write 2-3 paragraphs applying the 4 D's framework:
    • What did you delegate to AI vs. write yourself?
    • How did you describe what you wanted to the APIs?
    • How did you discern good responses from bad?
    • What diligence/iteration did your project require?

Key Takeaways

  • Environment setup is foundational - You now have a professional AI development environment that you'll use all summer
  • Both OpenAI and Anthropic have strengths - Understanding both APIs makes you flexible
  • The 4 D's are a mental model, not rules - Use them to reflect on your work, not as rigid constraints
  • API costs are real - Always set billing caps and monitor usage
  • Version control from day 1 - Git isn't optional in modern development
