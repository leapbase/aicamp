Core Implementation: Required Features

1. Streaming Responses

Display tokens as they arrive for better perceived performance. User can stop generation mid-stream.

2. Conversation History

Maintain context across multiple turns, store in memory, allow users to start new conversations

3. Tool Use (at least one)

Add a function the chatbot can call: calculator, web search, database lookup, or API call

4. System Prompt

Define personality and expertise, set clear boundaries, include examples of good interactions

How Retrieval Works: Embeddings

What Are Embeddings?

Vector representations of text. Similar concepts are close together in vector space. "dog" and "puppy" have similar embeddings; "dog" and "blockchain" have very different embeddings.

The Process

1. Split documents into chunks → 2. Generate embeddings for each chunk → 3. Store in vector database → 4. When user asks: embed question, find similar chunks, include in prompt

Build a "Chat with PDF" App

Tech Stack

  • Vector DB: Chroma
  • Embeddings: OpenAI API
  • PDF Parsing: PyPDF2
  • UI: Streamlit
  • LLM: Claude or GPT-4

Key Steps

  • Parse PDF into text
  • Split into chunks
  • Generate embeddings
  • Store in Chroma
  • Query interface

The Goal

Upload a PDF, then chat with its contents. Ask questions and get answers grounded in the document with page citations.

9 / 12

Implementation Steps

Step 1: PDF Processing

Parse PDF into text, split into chunks (500-1000 tokens each), handle overlapping chunks for context

Step 2: Create Embeddings

Generate embeddings for each chunk, store in Chroma with metadata (page number, chunk ID)

Step 3: Query Interface

User asks question, embed the question, retrieve top K similar chunks (K=3-5), show retrieved chunks in UI

Step 4: Augmented Generation

Build prompt with system instructions + retrieved chunks + user question, call LLM API, stream response

Key Takeaways

Purpose-Driven Chatbots Are More Valuable

Streaming Improves Perceived Performance

Deployment is Part of Development
